# mcp-audit v0.3.0

`v0.3.0` is the stable production hardening release. It turns team policy from a draft artifact into an enforceable local workflow and closes the loop around reviewed baselines, exceptions, guided setup, and feedback intake.

## Highlights

- Added enforced team policy checks through `mcp-audit policy check` and `mcp-audit scan --policy`.
- Added reviewed baseline hash verification so baseline changes can be treated as code-reviewable risk acceptance.
- Added time-bounded policy exceptions for approved finding fingerprints.
- Added `mcp-audit init --wizard --profile team` for guided team-ready setup.
- Added false-negative intake alongside the existing false-positive report path.
- Documented PyPI/TestPyPI publication status and current GitHub release artifact install path.

## Release artifacts

The GitHub release should include the CI-built source distribution and wheel:

- `mcp_audit-0.3.0.tar.gz`
- `mcp_audit-0.3.0-py3-none-any.whl`

Artifacts must come from the green CI run for the release commit.

## Verification

- `python3 -m pytest tests -q`
- `git diff --check`
- `PYTHONPATH=src python3 -m mcp_audit --version`
- `PYTHONPATH=src python3 -m mcp_audit.cli init --wizard --profile team`
- `PYTHONPATH=src python3 -m mcp_audit.cli policy check --policy .mcp-audit-policy.toml --profile team`
- GitHub CI on `main`

## Known limits

- PyPI/TestPyPI publication still requires package-index Trusted Publisher setup outside the repository.
- Homebrew packaging is documented as a next channel, but no tap formula is included in this release.
- Real-user feedback remains pending; public sample review and GitHub issue templates are the active substitute.
