# mcp-audit v0.4.0

`v0.4.0` improves adoption readiness after the first PyPI release. It keeps scanning local-first and conservative, while making onboarding, rule explanations, team governance, and Homebrew distribution clearer.

## Highlights

- Added `mcp-audit discover` to preview supported config paths without scanning files.
- Added next-step guidance to `mcp-audit doctor`.
- Expanded `mcp-audit explain` with acceptance and policy exception guidance.
- Added a team adoption playbook for CI gates, baseline review, exceptions, and PR review.
- Added Homebrew packaging guidance for `x-one-ai/tap/mcp-audit`.

## Release artifacts

The GitHub release should include the CI-built source distribution and wheel:

- `xone_mcp_audit-0.4.0.tar.gz`
- `xone_mcp_audit-0.4.0-py3-none-any.whl`

Artifacts must come from the green CI run for the release commit.

## Verification

- `python3 -m pytest tests -q`
- `git diff --check`
- `PYTHONPATH=src python3 -m mcp_audit --version`
- `PYTHONPATH=src python3 -m mcp_audit.cli discover`
- GitHub CI on `main`

## Known limits

- Real-user feedback remains the main product-quality gap and is tracked outside this release.
- Homebrew tap publication is a separate repository update after the PyPI package is available.
