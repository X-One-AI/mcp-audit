# mcp-audit v0.2.0

`v0.2.0` turns the first production baseline into a more realistic adoption release. It expands input coverage, adds real-world shaped rule evidence, starts distribution automation, and introduces the first team policy schema.

## Highlights

- Added a twelve-sample sanitized real-world corpus v2 covering MCP server examples, AI coding client configs, and high-risk filesystem, environment, Docker, and network patterns.
- Added rules for broad tool enablement, unpinned container images, and sensitive container environment passthrough.
- Improved unpinned package detection for nested MCP client formats, `bunx`, and floating `@latest` packages.
- Added an internal GitHub URL to sanitized fixture workflow for turning public config samples into safe fixtures.
- Added PyPI/TestPyPI Trusted Publishing workflow scaffolding.
- Added a team policy schema draft and parser, separate from rule profiles.
- Added a public sample review loop as the temporary substitute for unavailable 3-5 real-user configuration reviews.

## Release artifacts

The GitHub release should include the CI-built source distribution and wheel:

- `mcp_audit-0.2.0.tar.gz`
- `mcp_audit-0.2.0-py3-none-any.whl`

Artifacts must come from the green CI run for the release commit, not from an unverified local build.

## Verification

- `python3 -m pytest tests -q`
- `git diff --check`
- GitHub CI on `main`
- GitHub release assets uploaded from CI build artifacts

## Known Limits

- TestPyPI/PyPI publishing requires project-level Trusted Publisher configuration in the package indexes before the workflow can publish.
- Team policy schema is parsed and documented, but not yet enforced by the scan command.
- Real-user feedback is approximated with public sample review until 3-5 external users can run their own private configs.
