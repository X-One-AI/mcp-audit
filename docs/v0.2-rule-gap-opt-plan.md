# mcp-audit v0.2 Rule Gap OPT Plan

## 1. Product Manager

### Problem

The twelve-sample real-world corpus v2 exposed concrete gaps that limit production usefulness: broad tool enablement is invisible, Docker MCP images can float without review, and containers can receive sensitive host environment variables without literal secrets appearing in config.

### Goal

Turn the highest-confidence corpus v2 gaps into v0.2 release-candidate rules while preserving the product's bounded, local-first behavior.

### Non-Goals

- No PyPI or Homebrew release in this phase.
- No default scanning of home or global client paths.
- No recursive repository scanning.
- No hosted service, dashboard, or organization policy server.

### Requirements

| ID | Requirement | Acceptance |
| --- | --- | --- |
| V02-001 | Detect broad tool enablement | `enableAllTools: true`, `allowAllTools: true`, and `tools: "*"` emit `XONE008` |
| V02-002 | Detect unpinned Docker images | `docker run image`, `image:latest`, and untagged image references emit `XONE009` |
| V02-003 | Allow pinned Docker images | `image:1.2.3` and `image@sha256:...` do not emit `XONE009` |
| V02-004 | Detect sensitive Docker env passthrough | `docker run -e GITHUB_TOKEN` and similar names emit `XONE010` without exposing values |
| V02-005 | Preserve bounded discovery | Cline global paths are documented as explicit `--config`, not default discovery |
| V02-006 | Keep profile adoption sane | `starter` enables clear supply-chain checks; `balanced` and `team` enable all v0.2 rules |
| V02-007 | Keep rule documentation complete | Rule docs, QA plan, release checklist, README, and changelog mention v0.2 rules |

## 2. Architect

### Boundaries

- `src/mcp_audit/rules/tool_enablement.py`: broad tool enablement rule.
- `src/mcp_audit/rules/docker.py`: Docker image pinning and sensitive env passthrough rules.
- `src/mcp_audit/rules/registry.py`: profile membership and registration.
- `tests/fixtures/real-world-corpus-v2`: corpus expectations from public samples.
- `docs/rule-tuning-findings.md`: gaps closed or deferred.

### Data Flow

```text
parsed config object
  -> registered rule profile
  -> direct and nested config traversal
  -> findings with stable XONE IDs
  -> JSON / Markdown / SARIF reports
```

### Risk Controls

| Risk | Mitigation |
| --- | --- |
| Docker parsing overmatches CLI arguments | Only evaluate args for servers whose command is `docker`; skip flags and known docker subcommands |
| Env passthrough exposes secret values | Emit env variable names only, never values |
| New rules make first scan too noisy | Keep `XONE008` and `XONE010` out of `starter` |
| Cline docs imply home scanning by default | Keep explicit `--config` instructions only |
